# Grasp Pose 实时方案说明（Gemini 305）

## 1. 目标与范围

本方案用于在 Gemini 305 实时 RGBD 数据上估计抓取位姿，核心目标：

- 在不同视角（远近、左右、俯仰）下稳定检测托盘开口。
- 估计开口邻近平面（Opening Plane）与上平面（Top Plane）。
- 生成稳定的抓取坐标系，抑制方向跳动。
- 提供可观测的实时可视化（RGB + 高反差并排）。

当前脚本：

- `test_orbbec_realtime_estimate_grasp_pose_pipeline.py`

---

## 2. 算法思路（从目标到实现）

### 2.1 为什么分层做

直接从整帧找开口容易被背景/纹理/反光干扰，因此采用分层约束：

1. 先分割托盘（缩小搜索空间）。
2. 只在托盘前立面 ROI 找开口（细长暗槽）。
3. 以开口为几何锚点找邻近平面。
4. 由邻近平面约束上平面。
5. 由上平面点云法线稳定姿态方向。

### 2.2 开口检测核心

开口检测遵循 `estimate_grasp.py` 思路：

- 先在前立面 ROI 内生成候选（阈值 + 形态学 + 连通域）。
- 对候选做几何约束（细长比、尺寸、位置）。
- 用暗度/对比度/位置综合评分选择最优旋转框。
- 高反差灰度图只作为辅助评分，不替代主逻辑。

### 2.3 平面检测核心

- Opening Plane：从开口附近 ring 区域进行邻接生长（连续性 + 边缘约束），得到不规则平面区域。
- Top Plane：由 Opening Plane 邻接区域得到上平面候选，再拟合为规则四边形（旋转矩形）用于显示和约束。

### 2.4 姿态稳定策略

- `X`：由开口长边方向确定，参考方向 `(1,0,0)` 仅做正负号消歧。
- `Y`：由 Top Plane 法线主导，参考方向 `(0,-1,0)` 仅做正负号消歧。
- `Z`：由 `X/Y` 回构，保证右手系与正交稳定。

## 3. 实时管线过程

### 3.1 线程模型

- 主线程：采集、窗口显示、3D 刷新。
- 后台线程：重计算（托盘分割/开口/平面/姿态）。
- 单槽队列：避免计算阻塞实时预览。

### 3.2 每帧处理步骤

1. 采集 RGB + 点云。
2. 托盘分割（zero-shot 优先）。
3. 开口检测（前立面 ROI，旋转框）。
4. Opening Plane 分割（邻接生长）。
5. Top Plane 分割并拟合规则四边形。
6. 提取点云局部平面并求抓取位姿。
7. 更新 RGB/高反差并排预览与 3D 结果。

### 3.3 关键输入输出

- 输入：
  - RGB 图（`H x W x 3`）
  - 点云（`N x 3` 或 `N x 6`）
- 中间结果：
  - `tray_mask`
  - `opening.quad_uv`
  - `near_plane_mask`
  - `no_hole_mask`
  - `top_plane_quad_uv`
- 输出：
  - `grasp_point`
  - `rotation (3x3)`
  - `pre_grasp_point`

## 6. 调参与排障

### 6.1 开口检测漂移

优先检查：

1. 托盘分割是否稳定。
2. 前立面 ROI 是否覆盖到真实开口。
3. 开口候选几何约束是否过松/过紧。

### 6.2 Top Plane 过小或跳动

优先检查：

1. Opening Plane 是否邻接正确。
2. Top Plane 邻接区域大小是否足够。
3. Top Plane 点云法线拟合点数是否足够。

### 6.3 OpenCV 异常

- 脚本中已对 `cv2.error` 单独捕获。
- 若频繁出现，优先打印并检查当前 mask 是否为空或尺寸不一致。

---

## 7. 方案边界

- 强依赖托盘分割质量；分割失败会级联影响后续。
- 极端反光/遮挡下，开口候选排序可能失真。
- 当前为工程实时方案，非离线全局最优解。

---

## 8. 后续优化方向

1. 开口候选加入方向一致性约束（与前立面边方向平行）。
2. Top Plane 由规则四边形进一步做平面拟合重投影闭环。
3. 引入结果质量分数（置信度）用于上层决策。
